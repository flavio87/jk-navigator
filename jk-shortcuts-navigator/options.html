    Explanation

    • selectors: css selector for the list of links that should be iterated through.
    • paginator_selector_next: css selector for the link that goes to the next page of search results
    • paginator_selector_previous: css selector for the link that goes to the previous page of search results
    • allowSubdomains: wether or not docs.google.com should trigger the jk navigator with this selector (www is not treated as subdomain here and will always work)
    • search_selector: css selector for the search field
    • liveUpdateElement: for pages with ajax, we sometimes need to define a listener on the parent css element of the link list to know when they are all ready.
    • infiniteScroll: set to True if we have a page like facebook or quora that can dynamically load new links into the link list.
